GST Tax Rates API

Unlike legacy APIs that require multiple endpoints for searching HSN codes, semantic searches, and tax lookups, GST Accelerator does it all in a single POST request.

Our condition-aware engine aggregates CBIC Master Database rules, automatically evaluates conditionals (e.g. price thresholds, B2B, branded packaging), and returns the exact applicable rate with an audit trail.

Ideal for ERPs, checkout flows, and AI Agent integrations.

Base URL
https://gstaccelerator.in/v1

Authentication

All API requests must be authenticated using an API key. You can generate and manage your API keys from the Developer Dashboard.

Pass your API key in the X-API-Key HTTP header.

If you fail to provide a key, or provide an invalid key, the API will respond with a 401 Unauthorized error.

cURL Example
curl -X POST https://gstaccelerator.in/v1/lookup \
  -H "X-API-Key: gsta_live_*******************" \
  -H "Content-Type: application/json" \
  -d '{"description": "laptop"}'

Lookup GST Rate

POST /v1/lookup

Search for HSN/SAC codes and their active GST rates by providing a natural language description. Provide optional context variables to automatically resolve complex notification conditions.

Request Parameters

Parameter Type Description
description * string The name of the item or service (e.g., "cotton shirt", "AC repair"). Acts as a semantic search query.
sale_value_inr float Declared sale value in INR. Essential for resolving price-threshold conditions (e.g., footwear below ₹1000).
branded boolean Set to true if the product is pre-packaged and labelled (branded). Default is false.
b2b boolean Set to true if the buyer is a registered business (B2B). Default is false.
supply_type enum intrastate or interstate. Dictates how the recommended applicable_rate string is formatted.
Example Request payload
{
  "description": "split AC unit",
  "supply_type": "intrastate",
  "branded": true
}
Response (200 OK)
[
  {
    "hsn_code": "84151010",
    "description": "Air conditioning machines, split system",
    "tax_rates": {
      "igst": 18.0,
      "cgst": 9.0,
      "sgst": 9.0,
      "cess": 0.0,
      "total_intrastate": 18.0
    },
    "applicable_rate": {
      "intrastate": "CGST 9.0% + SGST 9.0% = 18.0%",
      "interstate": "IGST 18.0%"
    },
    "condition_applied": null,
    "confidence": 0.9,
    "notification_ref": "1/2017-Central Tax (Rate)",
    "needs_review": false
  }
]

Condition Resolver

The standout feature of GST Accelerator is its ability to resolve complex tax notification rules automatically.

For example, garments below ₹1,000 are taxed at 5%, while garments above ₹1,000 are taxed at 12%. When you provide sale_value_inr, our engine evaluates the condition and returns the correct rate.

  • If needs_review is true, human verification or more parameters are required to confirm the rate.
  • If condition_warning is not null, a condition was found but couldn't be resolved (e.g. you didn't provide sale_value_inr).
Example with condition applied
// Request with sale_value_inr
{
  "description": "cotton shirt",
  "sale_value_inr": 1500.00
}

// Response segment
{
  "tax_rates": { "igst": 12.0, ... },
  "condition_applied": "Price threshold (exceeding Rs. 1,000): PASSED — 1,500.00 > 1000",
  "condition_warning": null,
  "needs_review": false
}

Error Codes

The API uses standard HTTP status codes to indicate success or failure of requests.

Code Description
400 Bad Request: Invalid parameters were passed (e.g. missing description).
401 Unauthorized: The API key is missing, invalid, or revoked.
429 Too Many Requests: You have exceeded your tier's monthly limit.
500 Internal Server Error: Something went wrong on our end.
Error Example
{
  "detail": "Invalid API Key or Revoked."
}
